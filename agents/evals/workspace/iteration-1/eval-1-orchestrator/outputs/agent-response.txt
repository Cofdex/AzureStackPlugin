## Orchestrator Routing Decision

### 1. Workflow Classification: `hotfix`

Trigger criteria matched:
- Production system is actively down (orders failing for 3+ hours)
- Exact crash site identified: `src/orders/processor.py` line 87 — a single file
- Root cause is clear: `TypeError: cannot unpack non-iterable NoneType object`
  when the payment processor returns None on timeout instead of the expected tuple
- Scope is 1–2 files (add a None-guard before the unpack at line 87, possibly
  update the timeout call-site in the payment client as well)
- User explicitly signals urgency: "ASAP"

Rejected alternatives:
- `bugfix`: Would require Debug Agent → Planner → Implement → parallel review-gate.
  The crash location and cause are already fully specified; the full pipeline adds
  unnecessary latency for a production outage.
- `feature`: No new behavior requested.
- `refactor`/`security`: Not applicable.

### 2. Workflow ID

  hotfix-payment-timeout-001

Slug derivation: "payment" + "timeout" from "payment processor times out" — the
core of the failure. 1–2 words, lowercase, hyphenated.

### 4. Next Agent: Implement Agent

Per the hotfix workflow definition:

  Implement  →  fix/implementation-report.md
  Code Reviewer  →  fix/review-report.md  (sequential, not parallel)
  Secrets Scanner  →  fix/secrets-report.md  (automated, changed files only)
  SAST Scanner     →  fix/sast-report.md     (automated, changed files only)

The **Implement Agent** is spawned immediately with the following context:

  File:    src/orders/processor.py
  Line:    87
  Error:   TypeError: cannot unpack non-iterable NoneType object
  Cause:   The code unpacks the return value of the payment processor call
           without guarding against a None return on timeout.
  Fix:     Add an explicit None check (and likely a timeout/error check) before
           the unpack. Return or raise an appropriate domain error so the order
           is marked as "payment_pending" or "failed" rather than crashing.
  Output:  docs/workflows/hotfix-payment-timeout-001/fix/implementation-report.md

No TDD Suite, no Brainstorm, no Architecture Agent, no Reflection Agent for hotfix.
Automated Secrets + SAST scans are scoped to changed files only.
If the scan returns CRITICAL or HIGH severity, the Orchestrator escalates to the
user before committing.
